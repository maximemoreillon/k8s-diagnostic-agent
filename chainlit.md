# K8s Diagnostic Agent

Ask me anything about your Kubernetes cluster. I can help you diagnose issues such as crashing pods, image pull errors, resource exhaustion, and unschedulable workloads.

## What I can do

- List namespaces, pods, deployments, services, and nodes
- Describe individual resources in detail
- Fetch pod logs
- Identify common failure patterns: `CrashLoopBackOff`, `ImagePullBackOff`, `OOMKilled`, failed probes

## Example questions

- *Are there any issues in the `production` namespace?*
- *Why is pod `nginx-7d6b9c` crashing?*
- *Which nodes are under memory pressure?*
- *Show me the logs for the `api` pod in `staging`*
